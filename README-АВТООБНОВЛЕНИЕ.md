# 🌤️ Система автоматического обновления данных RP5

## 📖 Описание

Автоматическая система скачивания погодных данных с RP5.ru для всех городов с прогнозом погоды.

**Обновления:** каждые 3 часа (8 раз в день)  
**Стоимость:** 0₽ (полностью бесплатно)  
**Платформа:** GitHub Actions + Vercel

## 🎯 Что делает система

1. **Каждые 3 часа** запускается GitHub Actions workflow
2. **Скачивает** последние 7 дней погоды для всех станций
3. **Сохраняет** CSV файлы в `data/rp5-realtime/`
4. **Коммитит** изменения в репозиторий
5. **Vercel** автоматически деплоит обновлённый сайт

## 💻 Использование данных

### Node.js пример

```javascript
const fs = require('fs');
const wmoMapping = require('./data/wmo-mapping.js');

// Получить WMO ID для города
const cityId = 524901; // Москва
const wmoId = wmoMapping[cityId]; // '28440'

// Загрузить данные
const csvPath = `data/rp5-realtime/${wmoId}.csv`;
const data = fs.readFileSync(csvPath, 'utf-8');

// Парсить CSV
const lines = data.split('\n').slice(6); // Пропустить заголовок
const weather = lines.map(line => {
  const [datetime, temp, pressure, humidity, ...rest] = line.split(';');
  return {
    datetime,
    temperature: parseFloat(temp),
    pressure: parseFloat(pressure),
    humidity: parseFloat(humidity)
  };
});

console.log(weather);
```

### Express API пример

```javascript
app.get('/api/weather/:cityId', (req, res) => {
  const wmoId = wmoMapping[req.params.cityId];
  
  if (!wmoId) {
    return res.status(404).json({ error: 'City not found' });
  }
  
  const csvPath = `data/rp5-realtime/${wmoId}.csv`;
  
  if (!fs.existsSync(csvPath)) {
    return res.status(404).json({ error: 'Data not available' });
  }
  
  const data = parseCSV(fs.readFileSync(csvPath, 'utf-8'));
  res.json({ cityId: req.params.cityId, wmoId, data });
});
```

## 🛠️ Техническая информация

### Как работает скрипт

1. **Загружает список станций** из `data/wmo-mapping.js`
2. **Для каждой станции:**
   - Открывает страницу RP5 в headless Chrome
   - Переходит на вкладку "Скачать архив погоды"
   - Заполняет даты (последние 7 дней)
   - Выбирает формат CSV и кодировку UTF-8
   - Нажимает "Выбрать в файл"
   - Ждёт генерации архива
   - Скачивает и распаковывает CSV
3. **Сохраняет** файлы в `data/rp5-realtime/`

### Зависимости

- Python 3.11
- Selenium
- Chrome/ChromeDriver
- requests

Всё устанавливается автоматически в GitHub Actions!

### Формат CSV

```csv
Местное время;T;Po;P;Pa;U;DD;Ff;ff10;ff3;N;WW;W1;W2;Tn;Tx;Cl;Nh;H;Cm;Ch;VV;Td;RRR;tR;E;Tg;E';sss
02.04.2026 00:00;-2.1;748;747;-0.3;89;Юго-западный;3;5;7;100;Туман.;Туман.;Туман.;-3.2;-1.8;;;;;;;;;-3.7;0.0;6;;-4.2;;
```

Поля:
- `T` - температура (°C)
- `Po` - давление на уровне станции (мм рт.ст.)
- `P` - давление на уровне моря (мм рт.ст.)
- `U` - влажность (%)
- `DD` - направление ветра
- `Ff` - скорость ветра (м/с)
- `WW` - текущая погода
- `W1`, `W2` - прошедшая погода
- И т.д.

## 🔧 Настройка

### Изменить частоту обновлений

Редактируйте `.github/workflows/rp5-frequent.yml`:

```yaml
schedule:
  # Например, только 4 раза в день:
  - cron: '0 1 * * *'   # 06:00 Екатеринбург
  - cron: '0 7 * * *'   # 12:00 Екатеринбург
  - cron: '0 13 * * *'  # 18:00 Екатеринбург
  - cron: '0 19 * * *'  # 00:00 Екатеринбург
```

### Изменить период данных

Редактируйте `rp5-realtime-updater.py`:

```python
def get_date_range():
    """Возвращает диапазон дат для скачивания"""
    end_date = datetime.now()
    start_date = end_date - timedelta(days=30)  # Изменить на 30 дней
    
    return (
        start_date.strftime('%d.%m.%Y'),
        end_date.strftime('%d.%m.%Y')
    )
```

⚠️ **Внимание:** Больший период = дольше выполнение!

## ❓ FAQ

### Сколько это стоит?

**0₽** - полностью бесплатно! GitHub Actions даёт 2000 минут в месяц.

### Нужна ли кредитная карта?

**Нет!** Всё работает на бесплатных тарифах без карт.

### Что если GitHub Actions не хватит?

Используйте Oracle Cloud Always Free (тоже без карты) - см. `БЕСПЛАТНЫЕ-ВАРИАНТЫ.md`

### Можно ли запускать чаще?

Да, но учитывайте лимит 2000 минут/месяц. Каждый запуск ~5 минут.

### Данные обновляются на Vercel?

Да! Vercel автоматически деплоит при каждом push в main.

### Что если RP5 заблокирует?

Скрипт использует паузы между запросами (10 сек). Риск блокировки минимален.

### Можно ли скачивать параллельно?

Не рекомендуется - можете получить бан от RP5. Последовательное скачивание безопаснее.

### Workflow не запускается

**Решение:** Settings → Actions → General → Workflow permissions → "Read and write permissions"

### Файлы не коммитятся

**Проверьте `.gitignore`:**
```bash
cat .gitignore | grep rp5-realtime
```

GitHub Actions автоматически установит Chrome. Если ошибка повторяется - проверьте логи.

### Некоторые станции не скачались

Это нормально! Некоторые станции могут быть временно недоступны. Если >90% успешно - всё ОК.
