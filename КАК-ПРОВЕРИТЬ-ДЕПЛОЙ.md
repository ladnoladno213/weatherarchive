# Как проверить статус деплоя на Render

## Автоматический деплой

Render автоматически деплоит каждый push в main ветку:
1. Обнаруживает новый коммит
2. Запускает build (`npm install`)
3. Запускает сервер (`npm start`)

**Время:** обычно 2-3 минуты

## Проверка статуса

### Вариант 1: Через веб-интерфейс Render

1. Откройте https://dashboard.render.com
2. Выберите ваш сервис (weatherwebsite)
3. Смотрите раздел "Events" или "Logs"

**Статусы:**
- 🔵 Building - идёт сборка
- 🟢 Live - успешно задеплоено
- 🔴 Failed - ошибка

### Вариант 2: Проверка сайта

Просто откройте ваш сайт:
```
https://weatherarchive-pg4r.onrender.com
```

Если видите сайт - деплой успешен!

### Вариант 3: Проверка логов

На Render Dashboard → Logs можно увидеть:
```
Загружаем базу городов...
База городов загружена: 167410 записей
Сервер запущен: http://localhost:10000
```


Должна загрузиться без ошибок.

## Если что-то не работает

### Проблема: "Daily API request limit exceeded"

**Причина:** Лимит GeoNames ещё не сбросился (сбрасывается в полночь UTC)

**Решение:**
1. Подождать до завтра
2. Или создать свой GeoNames аккаунт (см. ПРОБЛЕМА-API-ЛИМИТЫ.md)

### Проблема: Сайт не загружается

**Причина:** Render "спит" (бесплатный план)

**Решение:** Подождите 30-60 секунд, сервис "проснётся"

### Проблема: Ошибка при деплое

**Причина:** Ошибка в коде или зависимостях

**Решение:**
1. Проверьте логи на Render Dashboard
2. Проверьте локально: `npm install && npm start`
3. Если нужно, откатите коммит: `git revert HEAD`

## Мониторинг

### Проверка здоровья сервиса

Render показывает:
- CPU usage
- Memory usage
- Response time
- Error rate

### Логи в реальном времени

Dashboard → Logs → Live logs

Можно увидеть:
- Запросы к сайту
- Ошибки
- Загрузку данных
