# RP5 Archive API - Прямые ссылки на архивы

## Обнаружение

Найдена прямая ссылка на архивные данные RP5:
```
https://ru1.rp5.ru/download/files.synop/28/28573.01.02.2005.01.04.2026.1.0.0.ru.utf8.00000000.csv.gz
```

Это архив метеостанции Ишим (WMO 28573) с 01.02.2005 по 01.04.2026.

## Структура URL

```
https://ru1.rp5.ru/download/files.synop/{FOLDER}/{WMO_ID}.{START_DATE}.{END_DATE}.1.0.0.ru.utf8.00000000.csv.gz
```

Где:
- `{FOLDER}` - первые 2 цифры WMO ID (например, `28` для `28573`)
- `{WMO_ID}` - полный WMO ID станции (5 цифр)
- `{START_DATE}` - дата начала в формате `DD.MM.YYYY`
- `{END_DATE}` - дата окончания в формате `DD.MM.YYYY`
- `.1.0.0.ru.utf8.00000000` - параметры (формат, язык, кодировка)
- `.csv.gz` - формат файла (gzip сжатый CSV)

## Ограничения

⚠️ **Важно:** Прямые ссылки возвращают `403 Forbidden` при программном доступе (через fetch/curl).

Возможные причины:
1. Требуется авторизация на сайте RP5
2. Проверка cookies/сессии
3. Защита от ботов
4. Проверка Referer заголовка

## Решение: Ручная загрузка через браузер

### Способ 1: Прямая ссылка в браузере

1. Откройте ссылку в браузере (предварительно зайдя на rp5.ru)
2. Файл автоматически скачается
3. Распакуйте .gz архив
4. Поместите CSV файл в папку `data/rp5-csv/`

### Способ 2: Через интерфейс RP5

См. инструкцию в `RP5-MANUAL-DOWNLOAD.md`

## Генератор ссылок

Создан модуль `rp5-archive-downloader.js` с функцией `buildRP5ArchiveURL()`:

```javascript
const { buildRP5ArchiveURL } = require('./rp5-archive-downloader.js');

// Построить URL для Москвы
const url = buildRP5ArchiveURL('27612', '2020-01-01', '2024-12-31');
console.log(url);
// https://ru1.rp5.ru/download/files.synop/27/27612.01.01.2020.31.12.2024.1.0.0.ru.utf8.00000000.csv.gz
```

## Автоматизация (будущее)

Для автоматической загрузки потребуется:

1. **Использовать Puppeteer/Playwright**
   - Запустить браузер
   - Зайти на rp5.ru
   - Получить cookies
   - Скачать файл с cookies

2. **Или использовать cookies из браузера**
   - Экспортировать cookies из браузера
   - Использовать их в fetch запросах

3. **Или найти официальный API**
   - Связаться с RP5
   - Получить API ключ

## Формат скачанных файлов

После скачивания и распаковки:

1. Файл в формате CSV с разделителем `;`
2. Кодировка UTF-8
3. Начинается с комментариев (строки с `#`)
4. Содержит все метеопараметры включая WW, W1, W2

Пример структуры:
```csv
#Метеостанция Ишим (28573)
#Период: 01.02.2005 - 01.04.2026
#
Местное время;T;Po;P;Pa;U;DD;Ff;ff10;ff3;N;WW;W1;W2;Cl;Nh;H;Cm;Ch;VV;Td;RRR;tR;E;Tg;E';sss
01.02.2005 00:00;-18.8;755.8;771.5;-0.3;85;Ветер, дующий с юго-юго-запада;3.0;;;100;...
```

## Сохранение файлов

Рекомендуемая структура:

```
data/rp5-csv/
  ├── 28573.2005-02-01.2026-04-01.ishim.utf8.csv
  ├── 27612.2020-01-01.2024-12-31.moscow.utf8.csv
  └── 26063.2020-01-01.2024-12-31.spb.utf8.csv
```

Формат имени: `{WMO_ID}.{START_DATE}.{END_DATE}.{CITY_NAME}.utf8.csv`

## Использование в коде

После загрузки файлов, они автоматически будут использоваться модулем `rp5-csv-loader.js`:

```javascript
const { loadRP5Data } = require('./rp5-csv-loader.js');

// Загрузить данные для Ишима
const data = await loadRP5Data('1505453', '2023-07-01', '2023-07-31');
```

Модуль автоматически найдет CSV файл в:
1. Корне проекта
2. Папке `data/rp5-csv/`

## Заключение

Хотя прямые ссылки на архивы RP5 существуют, программный доступ к ним ограничен. Рекомендуется:

1. Использовать ручную загрузку через браузер для приоритетных городов
2. Для остальных городов использовать Open-Meteo Archive API
3. В будущем рассмотреть автоматизацию через Puppeteer
