# Миграция на локальные данные (без GeoNames API)

## Проблема

GeoNames API имеет лимит 20,000 запросов/день. На публичном сайте это быстро исчерпывается.

## Решение

У вас уже есть локальная база **167,410 городов** в `data/cities.json`:
- Координаты (lat, lon)
- Названия на русском и английском
- Коды стран (cc)
- Коды регионов (adm1)
- Население (pop)

## Что сделано

### 1. Добавлена функция поиска ближайших городов

В `data/geo-db.js`:
```javascript
findNearest(lat, lon, limit = 1, maxDistance = 100)
```

Использует формулу Haversine для точного расчёта расстояния.

### 2. Создан модуль `weather-local.js`

Альтернативная версия `getCityHierarchy` без GeoNames API.

## Сравнение

| Функция | GeoNames API | Локальные данные |
|---------|--------------|------------------|
| Поиск города | ✅ | ✅ |
| Координаты | ✅ | ✅ |
| Код страны | ✅ | ✅ |
| Код региона | ✅ | ✅ (adm1) |
| Иерархия (admin1-4) | ✅ | ⚠️ Частично |
| Лимиты | ❌ 20k/день | ✅ Нет |
| Скорость | ⚠️ Сеть | ✅ Мгновенно |
| Русские названия | ✅ | ✅ |

## Что теряем

При переходе на локальные данные:
- Нет детальной иерархии (admin1, admin2, admin3, admin4)
- Нет автоматического перевода названий регионов

## Что получаем

- ✅ Нет лимитов API
- ✅ Мгновенная работа (без сетевых запросов)
- ✅ Работает офлайн
- ✅ Не зависит от внешних сервисов
- ✅ Бесплатно навсегда

## План миграции

### Вариант 1: Полная замена (рекомендуется)

Заменить `getCityHierarchy` на локальную версию:

```javascript
// В weather.js
const { getCityHierarchyLocal } = require('./weather-local');

// Заменить все вызовы:
// const hier = await getCityHierarchy(lat, lon);
const hier = await getCityHierarchyLocal(lat, lon);
```

### Вариант 2: Гибридный подход

Использовать локальные данные, fallback на GeoNames при ошибках:

```javascript
async function getCityHierarchyHybrid(lat, lon) {
  try {
    // Сначала пробуем локальные данные
    return await getCityHierarchyLocal(lat, lon);
  } catch (error) {
    // Fallback на GeoNames если нужно
    return await getCityHierarchy(lat, lon);
  }
}
```

### Вариант 3: Текущий (с кэшированием)

Оставить GeoNames, но с кэшированием на 24 часа (уже реализовано).

## Рекомендация

**Вариант 1 (полная замена)** - лучший выбор:
- Нет зависимости от внешних API
- Нет лимитов
- Быстрее работает
- Проще поддерживать

Единственный минус - нет детальной иерархии регионов, но для прогноза погоды это не критично.
