# Быстрая инструкция: Как найти WMO ID на RP5

## Метод 1: Через поиск на сайте

1. Откройте http://rp5.ru
2. В поиске введите название города (например, "Голышманово")
3. Выберите нужную метеостанцию из результатов
4. В адресной строке браузера найдите WMO ID

**Пример для Ишима:**
```
URL: http://rp5.ru/Погода_в_Ишиме,_аэропорт
Или: http://rp5.ru/archive.php?wmo_id=28573
WMO ID: 28573
```

## Метод 2: Через архив погоды

1. Откройте http://rp5.ru/archive.php
2. В поле "Населенный пункт" введите название города
3. Нажмите "Найти"
4. В результатах поиска будет показан WMO ID

## Метод 3: Через координаты

Если не можете найти город по названию:

1. Возьмите координаты из `russian-cities-list.json`
2. Откройте http://rp5.ru
3. Введите координаты в формате: `56.38611, 68.36667`
4. Выберите ближайшую станцию

## Что делать, если станция не найдена?

Если для города нет метеостанции:

1. **Найдите ближайший крупный город** с метеостанцией
2. **Используйте его WMO ID** (данные будут приблизительными)
3. **Или пропустите этот город** - система будет использовать модельные данные

## Формат WMO ID

- **Всегда 5 цифр**: например, `28573`
- **Без пробелов и дефисов**: `28573`, а не `28-573`
- **Без кавычек в коде**: `'28573'` (в строке)

## Проверка правильности

После добавления WMO ID:

1. Перезапустите сервер
2. Откройте `http://localhost:3000/archive?id=GEONAME_ID`
3. Проверьте консоль сервера:
   - ✅ `Downloading CSV from RP5...` - начало загрузки
   - ✅ `CSV downloaded successfully` - успешно
   - ❌ `Error downloading CSV` - ошибка (неправильный WMO ID)

## Частые ошибки

❌ **Неправильно:**
```javascript
'1505453': 28573,        // Без кавычек
'1505453': '28 573',     // С пробелом
'1505453': '28-573',     // С дефисом
```

✅ **Правильно:**
```javascript
'1505453': '28573',      // В кавычках, 5 цифр
```
