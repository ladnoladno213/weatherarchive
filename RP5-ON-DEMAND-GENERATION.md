# RP5 Archive: On-Demand Generation System

## Открытие

RP5 использует систему генерации архивов **по запросу** (on-demand):

- Архивные CSV файлы **не хранятся постоянно** на серверах
- Файлы **генерируются только после запроса** через веб-интерфейс
- После генерации файлы **кэшируются на определенное время**
- Прямые запросы к несгенерированным файлам возвращают **403 Forbidden**

## Как это работает

### Сценарий 1: Файл не сгенерирован
```
GET https://ru1.rp5.ru/download/files.synop/28/28573.01.01.2020.01.01.2026.csv.gz
→ 403 Forbidden (файл не существует в кэше)
```

### Сценарий 2: Файл сгенерирован через веб-интерфейс
```
1. Пользователь заходит на https://rp5.ru/archive.php?wmo_id=28573
2. Выбирает период и нажимает "Скачать архив"
3. RP5 генерирует файл и помещает в кэш
4. Файл доступен для прямого скачивания некоторое время

GET https://ru1.rp5.ru/download/files.synop/28/28573.01.01.2020.01.01.2026.csv.gz
→ 200 OK (файл в кэше)
```

## Почему Санкт-Петербург (26063) работает?

Санкт-Петербург - популярная станция, которую часто запрашивают:
- Много пользователей генерируют архивы для этого города
- Файлы почти всегда находятся в кэше
- Поэтому прямые запросы работают стабильно

## Стратегии работы с RP5

### Вариант 1: Ручная генерация (рекомендуется)
1. Открыть https://rp5.ru/archive.php?wmo_id=XXXXX
2. Выбрать нужный период
3. Нажать "Скачать архив"
4. Сразу после этого использовать Python downloader

**Плюсы:**
- Гарантированно работает
- Можно скачать любую станцию

**Минусы:**
- Требует ручного действия
- Не подходит для автоматизации

### Вариант 2: Автоматическая генерация через форму
Попытаться автоматизировать отправку формы на rp5.ru:

```python
# Псевдокод
1. POST на https://rp5.ru/responses/reFileSynopShort.php
2. Параметры: wmo_id, date_start, date_end, lang=ru
3. Дождаться генерации (может занять время)
4. Скачать сгенерированный файл
```

**Проблемы:**
- RP5 блокирует автоматические запросы (Error #FS001-0)
- Требуется эмуляция браузера (Selenium/Playwright)
- Сложная реализация

### Вариант 3: Использовать популярные станции
Работать только со станциями, которые часто запрашиваются:
- Санкт-Петербург (26063) ✓
- Москва (27612) - возможно
- Другие крупные города

**Плюсы:**
- Простая реализация
- Надежно работает

**Минусы:**
- Ограниченный набор станций
- Непредсказуемая доступность

### Вариант 4: Кэширование и переиспользование
1. Скачать архивы один раз (вручную или автоматически)
2. Сохранить локально
3. Обновлять только новые данные

**Плюсы:**
- Минимальная нагрузка на RP5
- Быстрый доступ к данным

**Минусы:**
- Требует хранилище
- Данные устаревают

## Рекомендации

### Для разработки и тестирования
Использовать Санкт-Петербург (26063) - стабильно работает

### Для продакшена
1. **Гибридный подход:**
   - Попытаться прямое скачивание
   - Если 403 → показать пользователю инструкцию по ручной генерации
   - После генерации → автоматически скачать

2. **Кэширование:**
   - Сохранять скачанные архивы локально
   - Обновлять только последние данные (инкрементально)

3. **Альтернативные источники:**
   - NOAA ISD (бесплатно, но другой формат)
   - Meteostat API (ограниченные данные)
   - Коммерческие API (платно)

## Технические детали

### Время жизни кэша
Неизвестно точно, но наблюдения показывают:
- Популярные станции: часы/дни
- Редкие станции: минуты/часы после генерации

### Серверы
RP5 использует несколько серверов:
- ru1.rp5.ru
- ru2.rp5.ru
- ru3.rp5.ru

Кэш может быть **не синхронизирован** между серверами!

### Формат URL
```
https://{server}.rp5.ru/download/files.synop/{prefix}/{wmo_id}.{start}.{end}.1.0.0.ru.utf8.00000000.csv.gz

Где:
- server: ru1, ru2, ru3
- prefix: первые 2 цифры WMO ID (28573 → 28)
- wmo_id: полный WMO ID
- start: DD.MM.YYYY
- end: DD.MM.YYYY
```

## Выводы

RP5 **не предназначен для массового автоматического скачивания**. Система on-demand generation - это защита от перегрузки серверов.

Для серьезного проекта рекомендуется:
1. Использовать RP5 для начальной загрузки данных (вручную)
2. Переключиться на NOAA ISD для регулярных обновлений
3. Или использовать коммерческие API для критически важных данных
