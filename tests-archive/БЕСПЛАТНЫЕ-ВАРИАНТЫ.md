# Полностью бесплатные варианты (без кредитных карт)

## Вариант 1: GitHub Actions ✅ РЕКОМЕНДУЮ

**Преимущества:**
- ✅ Полностью бесплатно
- ✅ Без кредитных карт
- ✅ 2000 минут в месяц (достаточно!)
- ✅ Автоматический деплой на Vercel

**Ограничения:**
- ⚠️ Cron работает с задержкой 5-15 минут
- ⚠️ Может пропустить запуск если GitHub перегружен

**Решение:**
Используйте `.github/workflows/rp5-frequent.yml` - уже создан!

**Расчет времени:**
- 8 запусков в день
- ~5 минут на запуск (все станции)
- 8 × 5 = 40 минут в день
- 40 × 30 = 1200 минут в месяц
- **Укладываемся в лимит 2000 минут!**

## Настройка GitHub Actions

### Шаг 1: Закоммитьте файлы

```bash
git add .github/workflows/rp5-frequent.yml
git add rp5-realtime-updater.py
git commit -m "Add RP5 auto-update"
git push
```

### Шаг 2: Проверьте Actions

1. Откройте GitHub → ваш репозиторий
2. Вкладка "Actions"
3. Должен появиться "RP5 Frequent Updates"

### Шаг 3: Ручной запуск для теста

1. Actions → "RP5 Frequent Updates"
2. "Run workflow" → "Run workflow"
3. Подождите 5-10 минут
4. Проверьте что файлы появились в `data/rp5-realtime/`

### Шаг 4: Настройте Vercel

Vercel автоматически деплоит при каждом push в main!

## Оптимизация для GitHub Actions

### Уменьшение времени выполнения

Если не укладываетесь в 2000 минут, можно:

1. **Скачивать меньше станций за раз:**
```python
# Разделить станции на группы
# Группа 1: 00:00, 06:00, 12:00, 18:00
# Группа 2: 03:00, 09:00, 15:00, 21:00
```

2. **Кэшировать Chrome:**
```yaml
- name: Cache Chrome
  uses: actions/cache@v3
  with:
    path: ~/.cache/selenium
    key: chrome-${{ runner.os }}
```

3. **Параллельное скачивание:**
```python
# В rp5-realtime-updater.py
from concurrent.futures import ThreadPoolExecutor
```
