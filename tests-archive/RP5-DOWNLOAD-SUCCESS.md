## Использование

### Быстрый старт

```python
from rp5_downloader import RP5Downloader

# Создаем загрузчик
downloader = RP5Downloader()

# Скачиваем Санкт-Петербург
file_path = downloader.download_station(
    station_id='26063',
    start_date='01.01.2024',
    end_date='31.12.2024'
)

if file_path:
    print(f"Успешно скачано: {file_path}")
    
    # Загружаем в DataFrame
    df = downloader.load_csv_to_dataframe(file_path)
    print(f"Загружено {len(df)} строк")
    
    # Подготавливаем для БД
    df_prepared = downloader.prepare_for_database(df, '26063')
    
    # Теперь можно загрузить в БД
    # df_prepared.to_sql('weather_data', engine, if_exists='append')

downloader.close()
```

## Технические детали

### Почему работает Санкт-Петербург?

1. **Cookies получены успешно:**
   ```
   PHPSESSID: 6be9a9a21e5ba03a2821544fd9bd0d09
   lang: ru
   ```

2. **Правильные заголовки:**
   - User-Agent: Chrome 120
   - Referer: https://rp5.ru/
   - Accept-Language: ru-RU,ru;q=0.9

3. **Session сохраняет состояние:**
   - Cookies передаются автоматически
   - Имитация браузера

4. **Альтернативные серверы:**
   - ru1 - работает для коротких периодов
   - ru2 - работает для длинных периодов
   - ru3 - не тестировался

### Почему не работают другие станции?

Возможные причины:
1. **Географическая блокировка** - разные серверы для разных регионов
2. **Rate limiting** - ограничение по IP адресу
3. **Популярность станции** - Санкт-Петербург более популярен
4. **Случайность** - защита работает непредсказуемо

## Рекомендации

### Для стабильной работы:

1. **Используйте проверенные станции:**
   - Санкт-Петербург (26063) - подтверждено работает
   - Протестируйте другие станции через `test-stations-availability.py`

2. **Оптимальные параметры:**
   - Период: не более 1 года за раз
   - Задержка: 2-3 секунды между запросами
   - Параллельность: 1-2 потока максимум

3. **Время скачивания:**
   - Ночное время (00:00 - 06:00 МСК) - меньше нагрузка
   - Выходные дни - меньше пользователей

4. **Обработка ошибок:**
   - Всегда проверяйте результат
   - Логируйте все попытки
   - Сохраняйте успешные файлы
