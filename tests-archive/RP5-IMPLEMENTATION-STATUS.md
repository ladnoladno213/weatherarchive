## Ограничения ⚠️

### 1. On-Demand Generation
**Проблема:** RP5 не хранит архивы постоянно. Файлы генерируются по запросу.

**Решение:**
- Использовать популярные станции (Санкт-Петербург)
- Генерировать архивы вручную через веб-интерфейс
- Кэшировать скачанные файлы локально

### 2. Доступность станций
**Проблема:** Большинство станций возвращают 403 Forbidden.

**Статус:**
- ✅ Санкт-Петербург (26063) - стабильно работает
- ❓ Москва (27612) - иногда доступна
- ❌ Ишим (28573) - требует генерации
- ❌ Новосибирск (29634) - требует генерации
- ❌ Екатеринбург (28698) - требует генерации

**Решение:**
1. Открыть https://rp5.ru/archive.php?wmo_id=XXXXX
2. Сгенерировать архив через веб-интерфейс
3. Скачать через Python downloader

### 3. Автоматизация
**Проблема:** RP5 блокирует автоматические запросы к форме генерации (Error #FS001-0).

**Возможные решения:**
- Selenium/Playwright для эмуляции браузера (сложно)
- Использовать только популярные станции (ограниченно)
- Переключиться на альтернативные источники (NOAA ISD)

downloader = RP5Downloader()

# Использовать Санкт-Петербург - стабильно работает

    station_id='26063',

## Известные проблемы

### 1. Datetime parsing warning
**Симптом:** `WARNING - Не удалось конвертировать datetime: time data "-11"`

**Причина:** Pandas пытается парсить температуру как datetime (ложное срабатывание)

**Статус:** Не критично, все строки успешно обрабатываются

### 2. Windows UTF-8 encoding
**Симптом:** Символы ✓/✗ не отображаются в консоли

**Решение:** Заменены на [OK]/[FAIL]

**Статус:** ✅ Исправлено

### 3. Python command not found
**Симптом:** `python: command not found` в PowerShell

**Причина:** Python установлен, но не в PATH или используется другое имя

**Решение:** Запускать через IDLE или добавить Python в PATH

**Статус:** Обходное решение работает
