# W1 и W2 ВСЕГДА заполнены (как в RP5)

## Проблема

В предыдущей реализации W1 и W2 могли оставаться пустыми, если не было "реальных явлений". Но в реальных данных RP5 **ВСЕГДА** заполнены все три столбца (WW, W1, W2), за исключением редких случаев.

## Правило RP5

**ВСЕГДА заполняем W1 и W2:**
- Если было реальное явление (гроза, ливень, снег, дождь) → показываем его
- Если не было реального явления → показываем облачность

## Реализация

### 1. Определение кода облачности

```javascript
const getCloudCode = (cloudcover) => {
  if (cloudcover == null) return 1; // По умолчанию "переменная облачность"
  if (cloudcover <= 50) return 0; // "Облака покрывали половину неба или менее"
  if (cloudcover < 100) return 1; // "Облака покрывали более половины неба в течение одной части..."
  return 2; // "Облака покрывали более половины неба в течение всего периода"
};
```

### 2. Сбор данных для W1 (3-6 часов)

```javascript
let w1BestCode = null;
let w1BestPriority = -1;
let w1CloudCodes = []; // Собираем коды облачности для усреднения

for (let j = 3; j <= 6 && i >= j; j++) {
  const prevCode = h.weathercode?.[i - j] ?? null;
  const prevCloud = h.cloudcover?.[i - j] ?? null;
  
  // Ищем явление с наивысшим приоритетом
  if (prevCode != null) {
    const wCode = wwToWCode(correctedCode);
    const priority = w1Priorities[wCode] || 0;
    
    if (priority > w1BestPriority) {
      w1BestCode = wCode;
      w1BestPriority = priority;
    }
  }
  
  // Собираем облачность
  if (prevCloud != null) {
    w1CloudCodes.push(getCloudCode(prevCloud));
  }
}

// Определяем средний код облачности
let w1CloudCode = null;
if (w1CloudCodes.length > 0) {
  const avgCloud = w1CloudCodes.reduce((a, b) => a + b, 0) / w1CloudCodes.length;
  w1CloudCode = Math.round(avgCloud);
}
```

### 3. Заполнение W1

```javascript
// Если нашли реальное явление - показываем его
// Если нет - показываем облачность
if (w1BestCode != null && w1BestPriority > 10) {
  W1 = W_PAST[w1BestCode] || null;
} else if (w1CloudCode != null) {
  W1 = W_PAST[w1CloudCode] || null;
}
```

### 4. Аналогично для W2 (6-12 часов)

```javascript
// Собираем данные за 6-12 часов
for (let j = 6; j <= 12 && i >= j; j++) {
  // ... аналогично W1
}

// Заполняем W2
if (w2BestCode != null && w2BestPriority > 10) {
  W2 = W_PAST[w2BestCode] || null;
} else if (w2CloudCode != null) {
  W2 = W_PAST[w2CloudCode] || null;
}
```

### 5. Правило "не дублировать"

```javascript
// Если W1 и W2 одинаковые, заменяем W2 на облачность
if (W1 && W2 && W1 === W2) {
  if (w2CloudCode != null) {
    W2 = W_PAST[w2CloudCode] || null;
  } else {
    W2 = null;
  }
}
```

## Примеры

### Пример 1: Гроза с ливнями

```
WW: "Гроза слабая или умеренная..."
W1: "Гроза (грозы) с осадками или без них."
W2: "Ливень (ливни)." (если были ливни)
    или "Облака покрывали более половины неба..." (если не было ливней)
```

### Пример 2: Ливень без грозы

```
WW: "Ливневый(ые) дождь(и) слабый(ые)..."
W1: "Ливень (ливни)."
W2: "Облака покрывали более половины неба..."
```

### Пример 3: Ясная погода

```
WW: null (или "Ясно")
W1: "Облака покрывали половину неба или менее..."
W2: "Облака покрывали половину неба или менее..."
```

### Пример 4: После грозы

```
WW: "Состояние неба в общем не изменилось."
W1: "Гроза (грозы) с осадками или без них."
W2: "Ливень (ливни)."
    или "Облака покрывали более половины неба..."
```

### Пример 5: Одинаковые явления в W1 и W2

```
Было:
W1: "Ливень (ливни)."
W2: "Ливень (ливни)."

Стало:
W1: "Ливень (ливни)."
W2: "Облака покрывали более половины неба..." (заменили на облачность)
```

## Приоритеты явлений

```javascript
const w1Priorities = {
  9: 100, // Гроза (самый высокий)
  8: 90,  // Ливень
  7: 80,  // Снег
  6: 70,  // Дождь
  5: 60,  // Морось
  4: 50,  // Туман
  3: 40,  // Метель
  2: 10,  // Облачность (низкий, но показываем)
  1: 10,  // Облачность
  0: 10,  // Облачность
};
```

## Результат

Теперь **ВСЕГДА** заполнены все три столбца:
- **WW**: текущая погода (или пусто, если ничего не происходит)
- **W1**: прошедшая погода за 3-6 часов (ВСЕГДА заполнено - явление или облачность)
- **W2**: прошедшая погода за 6-12 часов (ВСЕГДА заполнено - явление или облачность)

Это соответствует реальным данным RP5, где почти всегда заполнены все три столбца.

## Исключения

Единственный случай, когда W1/W2 могут быть пустыми - это самые первые часы архива (когда i < 3 для W1 или i < 6 для W2), потому что нет данных за предыдущие периоды.
