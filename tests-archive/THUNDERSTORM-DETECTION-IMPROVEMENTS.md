# Улучшение определения гроз в архиве погоды

## ✅ ТЕКУЩАЯ РЕАЛИЗАЦИЯ (Мультимодельный подход)

### Используется 10 глобальных моделей

Система параллельно запрашивает данные от всех доступных моделей и объединяет результаты:

| № | Модель | Страна | Разрешение | Доступна с | Покрытие |
|---|--------|--------|------------|------------|----------|
| 1 | **JMA GSM** | Япония | 55 км | 2016-01-01 | **10 лет** ✅ |
| 2 | **ECMWF IFS HRES** | Европа | 9 км | 2017-01-01 | 9 лет |
| 3 | **GFS** | США | 13 км | 2021-03-23 | 5 лет |
| 4 | **UKMO Global** | Великобритания | 10 км | 2022-03-01 | 4 года |
| 5 | **ECMWF IFS 0.4°** | Европа | 44 км | 2022-11-07 | 3+ года |
| 6 | **GEM Global** | Канада | 15 км | 2022-11-23 | 3+ года |
| 7 | **ICON Global** | Германия | 11 км | 2022-11-24 | 3+ года |
| 8 | **CMA GRAPES** | Китай | 15 км | 2023-12-31 | 2+ года |
| 9 | **ARPEGE World** | Франция | 25 км | 2024-01-02 | 2 года |
| 10 | **BOM ACCESS** | Австралия | 15 км | 2024-01-18 | 2 года |

### Алгоритм объединения данных

1. **Параллельный запрос** - все 10 моделей запрашиваются одновременно
2. **Два метода обнаружения:**
   - **Прямой:** weather_code 95-99 (работает для 8 моделей)
   - **Косвенный:** CAPE + осадки (для моделей без weather_code)
3. **Подсчёт согласия** - считается количество моделей, обнаруживших грозу
4. **Определение уверенности:**
   - **Очень высокая:** 5+ моделей согласны (≥50%)
   - **Высокая:** 3-4 модели согласны
   - **Средняя:** 2 модели согласны
   - **Низкая:** 1 модель обнаружила

### Критерии обнаружения по CAPE

Для моделей без прямого кода погоды (GFS, ECMWF IFS 0.4°):
- CAPE > 1500 J/kg + осадки > 5 мм → Гроза
- CAPE > 2500 J/kg + осадки > 2 мм → Гроза
- CAPE > 3500 J/kg → Гроза (экстремальная конвекция)

### Коды погоды для гроз (WMO):
- **95**: Гроза слабая или умеренная без града, но с дождём и/или снегом
- **96**: Гроза с небольшим градом
- **97**: Гроза сильная без града, но с дождём и/или снегом
- **98**: Гроза с пыльной бурей
- **99**: Гроза с сильным градом

### Уточнение интенсивности:
Для кода 95 используются дополнительные параметры:
- **CAPE > 1000 J/kg** или **осадки > 10 мм** → "Гроза умеренная"
- Иначе → "Гроза слабая"

## Варианты улучшения точности

### 1. Использование нескольких моделей (ансамблевый подход)

Можно запрашивать данные от нескольких моделей и считать грозу подтвержденной, если она есть в 2+ моделях:

#### Доступные модели в Historical Forecast API:

**Глобальные модели:**
- `icon_global` - DWD ICON (текущая, 11 км)
- `gfs_global` - NOAA GFS (13 км, с 2021-03-23)
- `ecmwf_ifs04` - ECMWF IFS (44 км, с 2022-11-07)
- `gem_global` - Canadian GEM (15 км, с 2022-11-23)
- `bom_access_global` - Australian BOM (15 км, с 2024-01-18)

**Региональные модели (высокое разрешение):**
- `icon_eu` - ICON Europe (7 км, Европа)
- `icon_d2` - ICON D2 (2 км, Центральная Европа)
- `arome_france` - Météo-France AROME (2.5 км, Франция)
- `harmonie_arome_europe` - HARMONIE AROME (5.5 км, Европа)

#### Пример реализации:

```javascript
// Запрашиваем данные от 3 моделей
const models = ['icon_global', 'gfs_global', 'ecmwf_ifs04'];
const results = await Promise.all(
  models.map(model => fetchThunderstormData(lat, lon, date, model))
);

// Считаем грозу подтвержденной если 2+ модели показывают грозу
const thunderstormCount = results.filter(r => r.hasThunderstorm).length;
if (thunderstormCount >= 2) {
  WW = 'Гроза (подтверждена несколькими моделями)';
}
```

### 2. Использование дополнительных параметров

Кроме `weather_code`, можно использовать:

- **CAPE** (Convective Available Potential Energy):
  - < 300 J/kg: слабая конвекция
  - 300-1000: умеренная
  - 1000-2500: сильная
  - \> 2500: экстремальная

- **Lifted Index** (если доступен):
  - \> 2: стабильная атмосфера
  - 0 to 2: слабо нестабильная
  - -2 to 0: умеренно нестабильная
  - < -2: очень нестабильная (грозы вероятны)

- **Precipitation intensity**:
  - Сильные осадки (> 10 мм/ч) часто связаны с грозами

### 3. Комбинирование с реальными наблюдениями

Для периодов где доступны METAR данные (аэропорты), можно:
1. Проверить METAR на наличие TS (thunderstorm)
2. Если есть TS в METAR, использовать это как подтверждение
3. Если нет METAR, использовать модельные данные

### 4. Временная корреляция

Грозы обычно длятся 30-60 минут. Можно:
- Проверять соседние часы
- Если гроза есть в соседних периодах, повышать уверенность

## Рекомендации по реализации

### Этап 1 (текущий): ✅ Выполнено
- Использование ICON Global
- Детальное описание типа грозы
- Уточнение интенсивности по CAPE

### Этап 2: Добавление GFS
- Запрашивать данные от GFS параллельно с ICON
- Если обе модели показывают грозу → высокая уверенность
- Если только одна → средняя уверенность

### Этап 3: Региональные модели
- Для Европы добавить ICON-EU или HARMONIE
- Для США добавить HRRR
- Использовать региональную модель как приоритетную если доступна

### Этап 4: Интеграция с METAR
- Для городов с аэропортами проверять METAR
- Использовать METAR как ground truth

## Пример кода для мультимодельного подхода

```javascript
async function detectThunderstorm(lat, lon, datetime) {
  const models = ['icon_global', 'gfs_global'];
  const results = [];
  
  for (const model of models) {
    const data = await fetchHistoricalForecast(lat, lon, datetime, model);
    const hasThunderstorm = data.weather_code >= 95 && data.weather_code <= 99;
    const cape = data.cape;
    
    results.push({
      model,
      hasThunderstorm,
      cape,
      confidence: hasThunderstorm ? (cape > 1000 ? 'high' : 'medium') : 'none'
    });
  }
  
  // Анализ результатов
  const thunderstormModels = results.filter(r => r.hasThunderstorm);
  
  if (thunderstormModels.length === 0) {
    return { hasThunderstorm: false };
  } else if (thunderstormModels.length === 1) {
    return {
      hasThunderstorm: true,
      description: 'Гроза (по данным одной модели)',
      confidence: 'medium'
    };
  } else {
    return {
      hasThunderstorm: true,
      description: 'Гроза (подтверждена несколькими моделями)',
      confidence: 'high'
    };
  }
}
```

## Ограничения

1. **Модельные данные не всегда точны** - модели могут пропускать локальные грозы или давать ложные срабатывания
2. **Разрешение моделей** - глобальные модели (11-13 км) могут пропускать мелкомасштабные грозы
3. **Доступность данных** - не все модели доступны для всех периодов и регионов
4. **Задержка данных** - архивные данные могут быть доступны с задержкой

## Выводы

Для максимальной точности рекомендуется:
1. Использовать 2-3 модели (ICON + GFS + региональная если доступна)
2. Учитывать CAPE и интенсивность осадков
3. Для аэропортов использовать METAR как приоритетный источник
4. Указывать уровень уверенности в определении грозы
