# Правило "Все три столбца" для WW/W1/W2

## Главное правило RP5

**Либо ВСЕ ТРИ столбца заполнены, либо ВСЕ ТРИ пустые**

- Если есть ХОТЬ ОДНО явление (в WW, W1 или W2) → заполняем ВСЕ ТРИ столбца
- Если НЕТ НИКАКИХ явлений → ВСЕ ТРИ пустые

## Примеры из реальных данных RP5 (2024 год)

### Пример 1: Все три пустые (1933 раза)
```
WW: (пусто)
W1: (пусто)
W2: (пусто)
```
**Условие**: Нет никаких явлений - ни сейчас, ни в прошлом

### Пример 2: Все три заполнены (137 раз)
```
WW: "Состояние неба в общем не изменилось."
W1: "Ливень (ливни)."
W2: "Облака покрывали более половины неба в течение всего соответствующего периода."
```
**Условие**: Был ливень в прошлом (W1), сейчас ничего не происходит

### Пример 3: Все три заполнены (66 раз)
```
WW: "Ливневый(ые) дождь(и) слабый(ые) в срок наблюдения или за последний час."
W1: "Ливень (ливни)."
W2: "Облака покрывали более половины неба в течение всего соответствующего периода."
```
**Условие**: Сейчас ливень (WW), был ливень в прошлом (W1)

### Пример 4: Все три заполнены (38 раз)
```
WW: "Состояние неба в общем не изменилось."
W1: "Гроза (грозы) с осадками или без них."
W2: "Ливень (ливни)."
```
**Условие**: Была гроза в прошлом (W1), сейчас ничего не происходит

## Реализация

### Шаг 1: Определяем WW, W1, W2 как обычно

```javascript
// Определяем WW из текущего weathercode
let WW = correctedWcode != null ? (WMO_WW[correctedWcode] || null) : null;

// Определяем W1 из диапазона 3-6 часов
// Если есть явление - показываем его, если нет - пока оставляем пустым
if (w1BestCode != null && w1BestPriority > 10) {
  W1 = W_PAST[w1BestCode] || null;
}

// Определяем W2 из диапазона 6-12 часов
// Если есть явление - показываем его, если нет - пока оставляем пустым
if (w2BestCode != null && w2BestPriority > 10) {
  W2 = W_PAST[w2BestCode] || null;
}
```

### Шаг 2: Проверяем, есть ли хоть одно явление

```javascript
const hasAnyPhenomenon = WW || 
                        (W1 && !W1.includes('Облака покрывали')) || 
                        (W2 && !W2.includes('Облака покрывали'));
```

### Шаг 3: Заполняем все три столбца или оставляем все пустыми

```javascript
if (hasAnyPhenomenon) {
  // Есть хоть одно явление - заполняем ВСЕ ТРИ столбца
  
  // Если WW пустой, но есть явления в прошлом - добавляем "Состояние неба..."
  if (!WW && (W1 || W2)) {
    const hasRealPhenomenonInPast = (W1 && !W1.includes('Облака покрывали')) || 
                                    (W2 && !W2.includes('Облака покрывали'));
    if (hasRealPhenomenonInPast && Cl >= 50) {
      WW = 'Состояние неба в общем не изменилось.';
    }
  }
  
  // Если W1 пустой - заполняем облачностью
  if (!W1 && w1CloudCode != null) {
    W1 = W_PAST[w1CloudCode] || null;
  }
  
  // Если W2 пустой - заполняем облачностью
  if (!W2 && w2CloudCode != null) {
    W2 = W_PAST[w2CloudCode] || null;
  }
} else {
  // Нет никаких явлений - оставляем ВСЕ ТРИ пустыми
  WW = null;
  W1 = null;
  W2 = null;
}
```

## Логика заполнения

### Случай 1: Есть текущее явление (WW заполнен)

```
WW: "Ливневый дождь слабый..."
W1: "Ливень (ливни)." (если был в диапазоне 3-6 часов)
    или "Облака покрывали..." (если не было явлений)
W2: "Облака покрывали..." (если W1 есть явление)
    или "Ливень (ливни)." (если W1 пустой, но был ливень в 6-12 часов)
```

### Случай 2: Нет текущего явления, но было в прошлом

```
WW: "Состояние неба в общем не изменилось."
W1: "Гроза (грозы)..." (если была гроза в 3-6 часов)
W2: "Ливень (ливни)." (если был ливень в 6-12 часов)
    или "Облака покрывали..." (если не было явлений)
```

### Случай 3: Нет никаких явлений

```
WW: (пусто)
W1: (пусто)
W2: (пусто)
```

## Что считается "явлением"?

**Реальные явления** (заполняем все три столбца):
- Гроза
- Ливень
- Снег
- Дождь
- Морось
- Туман
- Метель

**НЕ явления** (не заполняем столбцы):
- Облачность (коды 0-2 в W_PAST)
- Ясная погода (weathercode 0)

## Исключение

Единственный случай, когда может быть заполнен только WW:
```
WW: "  Диаметр изморозевого отложения составляет 2 мм."
W1: (пусто)
W2: (пусто)
```

Но API не может определять изморозь, поэтому этот случай не встречается в автоматически сгенерированных данных.

## Результат

После применения этого правила:
- ✅ Либо ВСЕ ТРИ столбца заполнены
- ✅ Либо ВСЕ ТРИ столбца пустые
- ❌ НЕ бывает ситуации, когда заполнен только один или два столбца

Это полностью соответствует реальным данным RP5.
