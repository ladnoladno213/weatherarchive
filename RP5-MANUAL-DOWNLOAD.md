# Ручная загрузка CSV файлов с RP5

## Проблема

Автоматическая загрузка CSV через API RP5 не работает - сервер возвращает ошибку `Error #FS001-0`. Это может быть связано с:
- Изменением API
- Требованием авторизации
- Защитой от ботов (капча)
- Ограничением частоты запросов

## Решение: Ручная загрузка

### Шаг 1: Найти WMO ID города

Используйте один из способов:

**Способ 1: Через веб-интерфейс**
1. Откройте страницу города на вашем сайте
2. WMO ID отображается в интерфейсе

**Способ 2: Через маппинг**
```javascript
const wmoMapping = require('./data/wmo-mapping.js');
const cityId = 524901; // ID Москвы в GeoNames
console.log('WMO ID:', wmoMapping[cityId]); // 27612
```

**Способ 3: Через скрипт**
```bash
node list-cities-with-ids.js
```

### Шаг 2: Открыть страницу архива RP5

URL формат:
```
http://rp5.ru/archive.php?wmo_id=XXXXX&lang=ru
```

Примеры:
- Ишим: http://rp5.ru/archive.php?wmo_id=28573&lang=ru
- Москва: http://rp5.ru/archive.php?wmo_id=27612&lang=ru
- Санкт-Петербург: http://rp5.ru/archive.php?wmo_id=26063&lang=ru

### Шаг 3: Заполнить форму

На странице архива:

1. **Период данных:**
   - Начальная дата: выберите нужную дату
   - Конечная дата: выберите нужную дату
   - Максимальный период: обычно 1-2 года за раз

2. **Параметры файла:**
   - ✅ Кодировка: UTF-8
   - ✅ Разделитель: точка с запятой (;)
   - ✅ Формат: только синоптические сроки (8 раз в сутки)

3. Нажать кнопку **"Скачать файл"**

### Шаг 4: Сохранить файл

Сохраните скачанный CSV файл в одну из папок:

**Вариант 1: Корень проекта** (для основного города)
```
28573.01.02.2005.31.03.2026.1.0.0.ru.utf8.00000000.csv
```

**Вариант 2: Папка data/rp5-csv/** (для дополнительных городов)
```
data/rp5-csv/27612.01.01.2020.31.12.2024.moscow.utf8.csv
```

Формат имени файла:
```
WMO_ID.DD.MM.YYYY.DD.MM.YYYY.city_name.utf8.csv
```

### Шаг 5: Проверить файл

Убедитесь, что файл:
- Начинается с символа `#` (комментарий)
- Содержит заголовки столбцов
- Имеет данные в формате CSV с разделителем `;`

Пример начала файла:
```csv
#Метеостанция Ишим (28573)
#Период: 01.02.2005 - 31.03.2026
#
Местное время;T;Po;P;Pa;U;DD;Ff;ff10;ff3;N;WW;W1;W2;Cl;Nh;H;Cm;Ch;VV;Td;RRR;tR;E;Tg;E';sss
01.02.2005 00:00;-18.8;755.8;771.5;-0.3;85;Ветер, дующий с юго-юго-запада;3.0;;;100;...
```

## Автоматизация (будущее)

Если RP5 API снова заработает, можно будет использовать:

```javascript
const { downloadRP5CSV } = require('./rp5-csv-downloader.js');

// Скачать CSV для города
const filePath = await downloadRP5CSV(
  '524901',      // geonameId (Москва)
  '2024-01-01',  // начальная дата
  '2024-12-31'   // конечная дата
);
```

## Альтернативы RP5

Если данные RP5 недоступны, используйте:

1. **Open-Meteo Archive API** (рекомендуется)
   - Бесплатно, без ограничений
   - Данные с 1940 года
   - Почасовые данные
   - API: https://archive-api.open-meteo.com/v1/archive

2. **NOAA ISD** (сырые данные)
   - Официальные данные метеостанций
   - Требует парсинга сложного формата
   - FTP: ftp://ftp.ncdc.noaa.gov/pub/data/noaa/

3. **Meteostat** (агрегированные данные)
   - API с историческими данными
   - Требует регистрацию
   - API: https://dev.meteostat.net/

См. подробности в `ALTERNATIVE-DATA-SOURCES.md`

## Поддержка

Если у вас возникли проблемы:
1. Проверьте, что WMO ID правильный
2. Убедитесь, что период не слишком большой (макс. 1-2 года)
3. Попробуйте другой браузер (иногда помогает)
4. Проверьте, что файл скачался полностью (размер > 0)
5. Откройте файл в текстовом редакторе и проверьте формат
