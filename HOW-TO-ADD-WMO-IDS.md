## Что нужно сделать

Вам нужно вручную найти WMO ID для городов на сайте http://rp5.ru и добавить их в файл `data/wmo-mapping.js`.

## Пошаговая инструкция

### Шаг 1: Выберите город из списка

Откройте файл `russian-cities-list.json` и найдите нужный город. Например:

```json
{
  "id": 1506422,
  "nameRu": "Голышманово",
  "nameEn": "Golyshmanovo",
  "population": 13424,
  "country": "RU",
  "lat": 56.38611,
  "lon": 68.36667
}
```

Запомните:
- **ID** (geonameId): `1506422`
- **Название**: Голышманово
- **Координаты**: 56.38611, 68.36667

### Шаг 2: Найдите WMO ID на RP5

1. Откройте http://rp5.ru
2. В поиске введите название города или координаты
3. Выберите ближайшую метеостанцию
4. В URL адресе найдите WMO ID

Например, для Ишима:
- URL: `http://rp5.ru/Погода_в_Ишиме,_аэропорт`
- Или в архиве: `http://rp5.ru/archive.php?wmo_id=28573`
- **WMO ID**: `28573`

### Шаг 3: Добавьте в маппинг

Откройте файл `data/wmo-mapping.js` и добавьте новую запись:

```javascript
const GEONAME_TO_WMO = {
  // Существующие записи...
  '1505453': '28573', // Ишим
  '524901': '27612',  // Москва
  
  // Добавьте новую запись:
  '1506422': 'WMO_ID_ЗДЕСЬ', // Голышманово
};

const GEONAME_TO_CITY = {
  // Существующие записи...
  '1505453': 'ishim',
  '524901': 'moscow',
  
  // Добавьте новую запись:
  '1506422': 'golyshmanovo',
};
```

### Шаг 4: Проверьте работу

1. Перезапустите сервер: `node server.js`
2. Откройте в браузере: `http://localhost:3000/archive?id=1506422`
3. Система автоматически скачает CSV с RP5 и покажет реальные данные

## Формат записи

### GEONAME_TO_WMO
```javascript
'geonameId': 'WMO_ID', // Название города
```

- **geonameId** - ID из `russian-cities-list.json` (поле `id`)
- **WMO_ID** - код метеостанции с RP5 (5 цифр)
- **Комментарий** - название города для удобства

### GEONAME_TO_CITY
```javascript
'geonameId': 'city-slug',
```

- **geonameId** - тот же ID
- **city-slug** - название города латиницей в нижнем регистре (используется в имени файла CSV)

1. **Начните с крупных городов** - у них точно есть метеостанции

## Что происходит после добавления

Когда вы добавляете WMO ID для города:

1. **Первый запрос**: Система скачивает CSV с RP5 (занимает 5-10 секунд)
2. **Кэширование**: CSV сохраняется в `data/rp5-csv/` на 24 часа
3. **Последующие запросы**: Данные берутся из кэша (мгновенно)
4. **Автообновление**: Через 24 часа CSV скачивается заново

## Структура CSV файла

Скачанный файл имеет формат:
```
data/rp5-csv/28573.2024-01-01.2024-12-31.ishim.utf8.csv
```

Где:
- `28573` - WMO ID
- `2024-01-01.2024-12-31` - период данных
- `ishim` - название города (из GEONAME_TO_CITY)
- `utf8` - кодировка

## Вопросы?

Если у вас возникли вопросы или проблемы:

1. Проверьте, что WMO ID правильный (5 цифр)
2. Убедитесь, что geonameId совпадает с ID из `russian-cities-list.json`
3. Перезапустите сервер после изменения `data/wmo-mapping.js`
4. Проверьте консоль сервера на наличие ошибок
