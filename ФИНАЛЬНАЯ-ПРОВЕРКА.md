# ✅ Финальная проверка перед запуском

## Проверьте эти пункты

### 1. Файлы на месте ✅

- [x] `.github/workflows/rp5-frequent.yml` - workflow для GitHub Actions
- [x] `rp5-realtime-updater.py` - скрипт скачивания
- [x] `data/wmo-mapping.js` - список станций

### 2. Структура директорий

Создайте папку для данных (если её нет):

```bash
mkdir -p data/rp5-realtime
```

### 3. Git настройки

Убедитесь что папка `data/rp5-realtime/` НЕ в `.gitignore`:

```bash
# Проверьте .gitignore
cat .gitignore | grep rp5-realtime

# Если нашлось - удалите эту строку!
```

### 4. GitHub настройки

После первого push проверьте:

1. **Actions включены:**
   - GitHub → Settings → Actions → General
   - "Allow all actions and reusable workflows" должно быть выбрано

2. **Права на запись:**
   - GitHub → Settings → Actions → General
   - Scroll down to "Workflow permissions"
   - Выберите "Read and write permissions"
   - ✅ Save

### 5. Vercel настройки

Убедитесь что Vercel подключен к вашему GitHub репозиторию:

1. Vercel Dashboard → ваш проект
2. Settings → Git
3. Должен быть подключен GitHub репозиторий
4. Auto-deploy должен быть включен для main/master ветки


## После push

1. Откройте GitHub → Actions
2. Найдите "RP5 Frequent Updates"
3. Нажмите "Run workflow"
4. Выберите ветку (main/master)
5. Нажмите зелёную кнопку "Run workflow"

## Использование данных на сайте

### В Node.js/Express:

```javascript
const fs = require('fs');
const path = require('path');

// Функция для загрузки данных станции
function loadStationData(wmoId) {
  const filePath = path.join(__dirname, 'data', 'rp5-realtime', `${wmoId}.csv`);
  
  if (!fs.existsSync(filePath)) {
    console.log(`Данные для станции ${wmoId} не найдены`);
    return null;
  }
  
  const content = fs.readFileSync(filePath, 'utf-8');
  const lines = content.split('\n');
  
  // Пропускаем заголовок (первые 6 строк)
  const dataLines = lines.slice(6);
  
  // Парсим CSV
  const data = dataLines
    .filter(line => line.trim())
    .map(line => {
      const parts = line.split(';');
      return {
        datetime: parts[0],
        temperature: parseFloat(parts[1]) || null,
        pressure: parseFloat(parts[2]) || null,
        humidity: parseFloat(parts[3]) || null,
        // ... остальные поля
      };
    });
  
  return data;
}

// Пример использования в роуте
app.get('/api/weather/:cityId', (req, res) => {
  const cityId = req.params.cityId;
  const wmoId = wmoMapping[cityId]; // из data/wmo-mapping.js
  
  if (!wmoId) {
    return res.status(404).json({ error: 'City not found' });
  }
  
  const weatherData = loadStationData(wmoId);
  
  if (!weatherData) {
    return res.status(404).json({ error: 'Weather data not available' });
  }
  
  res.json({
    cityId,
    wmoId,
    data: weatherData
  });
});
```

## Troubleshooting

### ❌ Workflow failed: "Permission denied"

**Решение:**
1. GitHub → Settings → Actions → General
2. Workflow permissions → "Read and write permissions"
3. Save

### ❌ Файлы не коммитятся

**Проверьте:**
```bash
# Убедитесь что папка не в .gitignore
cat .gitignore | grep rp5-realtime

# Если нашлось - удалите эту строку
```

### ❌ ChromeDriver error

**Это нормально для первого запуска!** GitHub Actions автоматически установит всё нужное.

Если ошибка повторяется - проверьте логи, возможно RP5 временно недоступен.

### ❌ Timeout (>6 часов)

Если выполнение занимает слишком долго:

1. Уменьшите `timeout-minutes` в workflow
2. Или разделите станции на группы

### ⚠️ Некоторые станции не скачались

**Это нормально!** Некоторые станции могут быть временно недоступны на RP5.

Если успешно скачалось >90% станций - всё ОК.

## Оптимизация

### Если не хватает лимита GitHub Actions

**Вариант 1:** Уменьшите частоту

Редактируйте `.github/workflows/rp5-frequent.yml`:

```yaml
schedule:
  # 4 раза в день вместо 8
  - cron: '0 1 * * *'   # 06:00 Екатеринбург
  - cron: '0 7 * * *'   # 12:00 Екатеринбург
  - cron: '0 13 * * *'  # 18:00 Екатеринбург
  - cron: '0 19 * * *'  # 00:00 Екатеринбург
```
