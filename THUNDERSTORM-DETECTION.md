# Проблема определения гроз в архиве

## Суть проблемы
В настоящем архиве погоды (RP5) отмечаются грозы, которые не отображаются в нашем архиве.

## Причина
**Open-Meteo использует модельные данные (ERA5 реанализ)**, а не данные с реальных метеостанций.

### Разница между источниками:

| Параметр | RP5 (SYNOP станции) | Open-Meteo (ERA5) |
|----------|---------------------|-------------------|
| Источник | Реальные наблюдения метеостанций | Модельный реанализ |
| Грозы | Фиксируются наблюдателем | Рассчитываются моделью |
| Точность для гроз | Высокая | Средняя/Низкая |
| Локальные явления | Фиксируются | Могут быть пропущены |

## Пример
**Малакка, 31 марта 2026:**
- RP5: Отмечена гроза
- Open-Meteo: Код 51 (морось), 63 (дождь) - грозы нет

## Почему так происходит?
1. **Грозы - локальное явление**: Могут быть в радиусе нескольких километров
2. **ERA5 - сетка 30км**: Модель усредняет данные по большой площади
3. **Конвективные процессы**: Сложно моделируются, особенно в тропиках

## Решения

### 1. Использовать RP5 (требует headless browser)
```javascript
const puppeteer = require('puppeteer');
// Медленно, ресурсоёмко, может быть заблокировано
```

### 2. Использовать METAR/SYNOP API
Найти API который предоставляет данные с реальных станций:
- NOAA API
- Aviation Weather Center
- Ogimet (требует парсинга)

### 3. Комбинированный подход
- Open-Meteo для основных параметров (быстро)
- METAR для гроз и особых явлений (если доступно)

### 4. Добавить примечание (текущее решение)
Указать пользователю что данные модельные и грозы могут быть не все зафиксированы.

## Рекомендация
Для продакшена оставить Open-Meteo с примечанием об ограничениях. Если критично нужны точные данные о грозах - рассмотреть интеграцию с METAR API или RP5 через Puppeteer (с кэшированием).
