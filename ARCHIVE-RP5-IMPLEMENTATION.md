# Применение правил RP5 к архиву погоды

## Выполненные изменения

### 1. Обновлены описания погоды в `data/wmo-codes.js`

Все описания WW (текущая погода) и W_PAST (прошедшая погода) обновлены на основе реальных данных из CSV файла RP5 (61,549 записей, 2005-2026).

- Код 28: "Туман или ледяной туман." (с точкой)

### 2. Обновлена логика заполнения W1 и W2 в `server.js`

**Правило из RP5: показываем только реальные явления**

```javascript
// W1 и W2 - прошедшая погода (за 3-6 и 6-12 часов)
// Правило из RP5: показываем только реальные явления, не облачность
let W1 = null;
let W2 = null;

// Определяем W1 (за 3-6 часов назад)
if (prevCode1 != null) {
  const w1Code = wwToWCode(prevCode1);
  const w1Text = W_PAST[w1Code] || null;
  // Показываем только если это реальное явление (не облачность)
  if (w1Text && !w1Text.includes('Облака покрывали')) {
    W1 = w1Text;
  }
}

// Определяем W2 (за 6-12 часов назад)
if (prevCode2 != null) {
  const w2Code = wwToWCode(prevCode2);
  const w2Text = W_PAST[w2Code] || null;
  // Показываем только если это реальное явление (не облачность)
  if (w2Text && !w2Text.includes('Облака покрывали')) {
    W2 = w2Text;
  }
}

// Правило "не дублировать": если W1 и W2 одинаковые, убираем W2
if (W1 && W2 && W1 === W2) {
  W2 = null;
}
```

**Что изменилось:**
- W1 и W2 теперь показывают ТОЛЬКО реальные погодные явления (дождь, снег, ливни, грозы, туман, морось, метель)
- Информация об облачности ("Облака покрывали...") НЕ показывается в W1/W2
- Если W1 и W2 одинаковые, W2 убирается (правило "не дублировать")

### 3. Обновлена логика для гроз

**Правило из RP5: точные формулировки**

```javascript
if (analysis.hasThunderstorm) {
  // Обновляем WW - используем описание грозы из анализа
  WW = analysis.description;
  
  // Обновляем W1 - гроза в текущем периоде или за последние 3-6 часов
  // Правило из RP5: если гроза, то W1 = "Гроза (грозы) с осадками или без них."
  // Если были ливни, то W1 = "Ливень (ливни)."
  const hasShowers = wcode >= 80 && wcode <= 86;
  if (hasShowers) {
    W1 = 'Ливень (ливни).';
  } else {
    W1 = 'Гроза (грозы) с осадками или без них.';
  }
  
  // W2 - проверяем что было 6 часов назад
  if (hadThunderstormLast6Hours) {
    W2 = 'Гроза (грозы) с осадками или без них.';
  } else if (hadShowersLast6Hours) {
    W2 = 'Ливень (ливни).';
  }
  
  // Правило "не дублировать": если W1 и W2 одинаковые, убираем W2
  if (W1 && W2 && W1 === W2) {
    W2 = null;
  }
}
```

**Что изменилось:**
- WW теперь содержит только описание грозы (без дублирования информации об облачности)
- W1 = "Гроза (грозы) с осадками или без них." (точная формулировка из RP5)
- Если были ливни, W1 = "Ливень (ливни)." (приоритет ливней)
- W2 показывает грозу или ливни 6 часов назад
- Правило "не дублировать" применяется к W1 и W2

### 4. Упрощена дедупликация

**Удалены:**
- Дедупликация W1 и W2 (теперь показываем все реальные явления)
- Специальная обработка "Состояние неба не изменялось"
- Переменные `lastW1`, `lastW2`, `w1RepeatCount`, `w2RepeatCount`, `lastSkyState`, `skyStateRepeatCount`

**Оставлена:**
- Дедупликация WW для неважных явлений (максимум 2 раза подряд)
- Важные явления (грозы, осадки, туман, дымка) показываются без ограничений

```javascript
// Дедупликация WW - показываем максимум 2 раза подряд (кроме важных явлений)
// Важные явления (грозы, осадки, туман, дымка) показываем без ограничений
const isImportantPhenomenon = WW && (
  WW.includes('Гроза') || WW.includes('гроза') ||
  WW.includes('Туман') || WW.includes('туман') ||
  WW.includes('Дымка') || WW.includes('дымка') ||
  WW.includes('Дождь') || WW.includes('дождь') ||
  WW.includes('Снег') || WW.includes('снег') ||
  WW.includes('Морось') || WW.includes('морось') ||
  WW.includes('Ливневый') || WW.includes('ливневый')
);

if (!isImportantPhenomenon && WW === lastWW && WW !== null) {
  wwRepeatCount++;
  if (wwRepeatCount > 2) {
    WW = null; // Скрываем после 2 повторений
  }
} else {
  wwRepeatCount = 1;
  lastWW = WW;
}
```

## Результаты

### Что улучшилось:

1. **Точные формулировки** - все описания теперь соответствуют реальным данным RP5
2. **Правило "не дублировать"** - информация не повторяется в WW/W1/W2
3. **Только реальные явления** - W1 и W2 показывают только дождь, снег, ливни, грозы, туман, морось, метель (не облачность)
4. **Правильные приоритеты** - Гроза > Ливень > Дождь > Снег > Морось > Туман > Облачность
5. **Временные рамки** - WW (последний час), W1 (3-6 часов), W2 (6-12 часов)

**Гроза с ливнем:**
