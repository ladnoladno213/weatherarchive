# Покрытие мультимодельной системы обнаружения гроз

## Обзор

Система использует **10 глобальных метеорологических моделей** для максимально точного обнаружения гроз в архивных данных. Данные от всех доступных моделей объединяются для повышения надёжности.

## Методы обнаружения гроз

### Метод 1: По коду погоды (weather_code)
Модели с прямым кодом погоды:
- JMA GSM ✅
- ECMWF IFS HRES ✅
- UKMO Global ✅
- GEM Global ✅
- ICON Global ✅
- CMA GRAPES ✅
- ARPEGE World ✅
- BOM ACCESS ✅

**Коды гроз:**
- 95 = Гроза слабая/умеренная
- 96 = Гроза с небольшим градом
- 97 = Гроза сильная
- 98 = Гроза с пыльной бурей
- 99 = Гроза с сильным градом

### Метод 2: По CAPE и осадкам
Для моделей без weather_code (GFS, ECMWF IFS 0.4°):

**Критерии обнаружения:**
1. CAPE > 1500 J/kg + осадки > 5 мм → Гроза
2. CAPE > 2500 J/kg + осадки > 2 мм → Гроза
3. CAPE > 3500 J/kg → Гроза (экстремальная конвекция)

**Модели с CAPE:**
- GFS ✅
- UKMO Global ✅
- GEM Global ✅
- ICON Global ✅
- CMA GRAPES ✅
- ARPEGE World ✅

## Алгоритм объединения данных

### Шаг 1: Сбор данных
Система параллельно запрашивает данные от всех 10 моделей для указанного периода.

### Шаг 2: Анализ каждой модели
Для каждого часа проверяется:
- Наличие кода погоды 95-99 (прямое обнаружение)
- Значения CAPE и осадков (косвенное обнаружение)

### Шаг 3: Подсчёт согласия моделей
Считается количество моделей, обнаруживших грозу в данный час.

### Шаг 4: Определение уверенности
- **Очень высокая** (very_high): 5+ моделей согласны (≥50%)
- **Высокая** (high): 3-4 модели согласны
- **Средняя** (medium): 2 модели согласны
- **Низкая** (low): 1 модель обнаружила

### Шаг 5: Формирование описания
Используются максимальные значения:
- Максимальный код погоды среди всех моделей
- Максимальный CAPE
- Максимальные осадки

**Описания:**
- "Гроза слабая" - код 95, CAPE < 1000, осадки < 10 мм
- "Гроза умеренная" - код 95, CAPE > 1000 или осадки > 10 мм
- "Гроза с небольшим градом" - код 96
- "Гроза сильная" - код 97
- "Гроза с пыльной бурей" - код 98
- "Гроза с сильным градом" - код 99

## Преимущества мультимодельного подхода

### 1. Повышенная точность
Объединение данных от 10 моделей значительно снижает количество ложных срабатываний и пропусков.

### 2. Глубокая история
Благодаря JMA GSM доступны данные с 2016 года (10 лет архива).

### 3. Глобальное покрытие
Все модели имеют глобальное покрытие, работают для любой точки мира.

### 4. Устойчивость к сбоям
Если одна модель недоступна, система продолжает работать с остальными.

### 5. Разнообразие методов
Комбинация прямого обнаружения (weather_code) и косвенного (CAPE) повышает надёжность.

## Технические детали

### API
```
https://historical-forecast-api.open-meteo.com/v1/forecast
```

### Параметры запроса
```
latitude, longitude, start_date, end_date
hourly=weather_code,cape,precipitation,showers
timezone=auto
models=<model_name>
```

### Таймауты
- Запрос к каждой модели: 15 секунд
- Все запросы выполняются параллельно

### Обработка ошибок
- HTTP 429 (Too Many Requests) - модель пропускается
- Timeout - модель пропускается
- Нет данных - модель пропускается
- Система продолжает работу с доступными моделями
