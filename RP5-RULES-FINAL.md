# Окончательные правила заполнения WW/W1/W2 из RP5

## Анализ реальных данных за 2024 год

### Правило для W2:

**W2 показывает облачность, ЕСЛИ в W1 есть реальное явление**

Примеры из CSV:

```
31.12.2024 20:00:
WW: "Слабый или умеренный поземок в целом низкий..."
W1: "Метель"
W2: "Облака покрывали более половины неба в течение всего соответствующего периода."
```

```
31.12.2024 08:00:
WW: "Слабый или умеренный поземок..."
W1: "Метель"
W2: "Облака покрывали более половины неба..."
```

```
31.12.2024 05:00:
WW: "Слабый или умеренный поземок..."
W1: "Снег и/или другие виды твердых осадков"
W2: "Метель"
```

### Логика:

1. **W1** - показываем ТОЛЬКО реальные явления (дождь, снег, ливень, гроза, туман, морось, метель)
2. **W2** - показываем:
   - Облачность, ЕСЛИ в W1 есть реальное явление
   - Реальное явление, ЕСЛИ в W1 НЕТ реального явления
3. **Правило "не дублировать"**: если W1 === W2, то W2 = null

## Реализация в server.js

```javascript
// W1 и W2 - прошедшая погода (за 3-6 и 6-12 часов)
// Правило из RP5 (из анализа реальных данных):
// W1 - показываем реальные явления (дождь, снег, ливень, гроза, туман, морось, метель)
// W2 - показываем облачность, ЕСЛИ в W1 было реальное явление, иначе тоже реальное явление
let W1 = null;
let W2 = null;

// Определяем W1 (за 3-6 часов назад)
if (prevCode1 != null) {
  const w1Code = wwToWCode(prevCode1);
  const w1Text = W_PAST[w1Code] || null;
  // W1 показываем только реальные явления (не облачность)
  if (w1Text && !w1Text.includes('Облака покрывали')) {
    W1 = w1Text;
  }
}

// Определяем W2 (за 6-12 часов назад)
if (prevCode2 != null) {
  const w2Code = wwToWCode(prevCode2);
  const w2Text = W_PAST[w2Code] || null;
  
  // Правило из RP5: если в W1 есть реальное явление, то в W2 можно показать облачность
  // Если в W1 нет явления, то в W2 показываем только реальное явление
  if (W1) {
    // В W1 есть явление - можем показать облачность в W2
    W2 = w2Text;
  } else {
    // В W1 нет явления - показываем только реальное явление в W2
    if (w2Text && !w2Text.includes('Облака покрывали')) {
      W2 = w2Text;
    }
  }
}

// Правило "не дублировать": если W1 и W2 одинаковые, убираем W2
if (W1 && W2 && W1 === W2) {
  W2 = null;
}
```

## Проблема с различением мороси и ливня

Open-Meteo API часто путает морось (коды 50-55) и ливни (коды 80-82).

**НЕ МОЖЕМ** различить по количеству осадков, потому что:
- Морось может быть интенсивной
- Ливень может быть слабым
- Количество осадков зависит от продолжительности

**РЕШЕНИЕ**: Оставить как есть, использовать weathercode от Open-Meteo.
Для точных описаний пользователь может редактировать вручную через `/archive/edit`.

## Что НЕ НУЖНО делать

❌ Не пытаться различать морось и ливень по количеству осадков
❌ Не пытаться различать по интенсивности
❌ Не пытаться различать по температуре

## Что НУЖНО сделать

✅ Применить правило для W2 (показывать облачность, если в W1 есть явление)
✅ Оставить формулировки из RP5 в `data/wmo-codes.js`
✅ Использовать weathercode от Open-Meteo как есть
✅ Предоставить интерфейс редактирования для точных описаний

## Итоговая логика

1. **WW** - текущая погода (последний час) - берем из weathercode
2. **W1** - прошедшая погода (3-6 часов) - ТОЛЬКО реальные явления
3. **W2** - прошедшая погода (6-12 часов):
   - Если W1 есть → показываем облачность или реальное явление
   - Если W1 нет → показываем ТОЛЬКО реальное явление
4. **Дедупликация**: если W1 === W2, то W2 = null

Это соответствует реальным данным RP5!
