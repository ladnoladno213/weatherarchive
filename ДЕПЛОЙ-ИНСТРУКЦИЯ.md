# Инструкция по деплою WeatherWebsite

Ваш сайт - это Node.js приложение на Express.js. Есть несколько вариантов бесплатного хостинга.

## 🚀 Рекомендуемые платформы

### 1. Render.com (Рекомендуется)

**Плюсы:**
- Бесплатный tier (750 часов/месяц)
- Автоматический деплой из GitHub
- Простая настройка
- Поддержка переменных окружения

**Шаги:**

1. Зарегистрируйтесь на https://render.com
2. Подключите GitHub аккаунт
3. Нажмите "New +" → "Web Service"
4. Выберите репозиторий `weatherarchive`
5. Настройки:
   - Name: `weatherwebsite`
   - Environment: `Node`
   - Build Command: `npm install`
   - Start Command: `npm start`
   - Plan: `Free`
6. Добавьте переменные окружения (Environment Variables):
   - `NODE_ENV` = `production`
   - `EDIT_PASSWORD` = `ваш_пароль` (для редактирования архива)
7. Нажмите "Create Web Service"

**Готово!** Сайт будет доступен по адресу `https://weatherwebsite.onrender.com`

**Важно:** На бесплатном плане сервис "засыпает" после 15 минут неактивности. Первый запрос после сна займёт ~30 секунд.

---

### 2. Railway.app

**Плюсы:**
- $5 бесплатных кредитов/месяц
- Быстрый деплой
- Не засыпает

**Шаги:**

1. Зарегистрируйтесь на https://railway.app
2. Нажмите "New Project" → "Deploy from GitHub repo"
3. Выберите репозиторий `weatherarchive`
4. Railway автоматически определит Node.js проект
5. Добавьте переменные окружения:
   - `EDIT_PASSWORD` = `ваш_пароль`
6. Нажмите "Deploy"

**Готово!** Сайт будет доступен по сгенерированному URL.

---

### 3. Vercel

**Плюсы:**
- Бесплатный tier
- Очень быстрый CDN
- Автоматический HTTPS

**Шаги:**

1. Зарегистрируйтесь на https://vercel.com
2. Нажмите "Add New..." → "Project"
3. Импортируйте репозиторий `weatherarchive`
4. Vercel автоматически определит настройки (используется `vercel.json`)
5. Добавьте переменные окружения:
   - `EDIT_PASSWORD` = `ваш_пароль`
6. Нажмите "Deploy"

**Готово!** Сайт будет доступен по адресу `https://weatherwebsite.vercel.app`

## 📋 Подготовка проекта

Все необходимые конфигурационные файлы уже созданы:

- ✅ `vercel.json` - конфигурация для Vercel
- ✅ `render.yaml` - конфигурация для Render
- ✅ `railway.toml` - конфигурация для Railway
- ✅ `Procfile` - универсальный файл запуска
- ✅ `package.json` - зависимости и скрипты

## 🔧 Переменные окружения

Обязательные:
- `PORT` - порт сервера (автоматически устанавливается платформой)
- `NODE_ENV` - `production` для продакшена

Опциональные:
- `EDIT_PASSWORD` - пароль для редактирования архива (по умолчанию: `admin123`)

## 📊 Требования к ресурсам

- Node.js: 18.x или выше
- RAM: ~512 MB
- Диск: ~200 MB (с данными)

## 🔍 Проверка перед деплоем

Локальный тест:

```bash
npm install
npm start
```

Откройте http://localhost:3000

## ⚠️ Важные замечания

1. **Данные RP5**: Файлы в `data/rp5-realtime/` обновляются через GitHub Actions каждые 3 часа. На хостинге они будут доступны только после git pull.

2. **Большие файлы**: Файл `data/alternateNamesV2.txt` исключён из репозитория (.gitignore). Если он нужен, загрузите отдельно.

3. **Puppeteer**: Библиотека `puppeteer` занимает ~300 MB. Если не используется на продакшене, можно убрать из зависимостей.

4. **Автоматический деплой**: Все платформы поддерживают автоматический деплой при push в main ветку.
